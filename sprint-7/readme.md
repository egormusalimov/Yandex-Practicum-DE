# Организация Data Lake

## Описание

Используя принципы ООП, нужно обновить структуру данных Data Lake:

- Создать витрины данных **по пользователям**
- Создать витрины данных **по географическим зонам**
- Наполнить эти витрины обновляемыми данными
- Настроить автоматическое обновление витрин
- Построить итоговую витрину данных **с рекомендациями**

## Этапы

1. Обновить структуру Data Lake
2. Создать витрину в разрезе пользователей
3. Создать витрину в разрезе географических зон
4. Построить витрину рекомендаций друзей
5. Автоматизировать обновление витрин

---

## Описание рабочих файлов

**Папка `src/scripts`:**

- `1_geography.py` — скрипт, вычисляющий геопозицию пользователей.
- `2_zones.py` — скрипт, вычисляющий географические зоны.
- `3_mart.py` — скрипт c рекомендациями.

**Папка `src/dags`:**

- `dag_mart.py` — DAG, который обновляет геопозицию пользователей, географические зоны и рекомендации.
